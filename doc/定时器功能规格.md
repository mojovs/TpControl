# 定时器功能规格

> 采访日期: 2026/06/18
> 采访对象: 用户 (Meng)

## 1. 启动时机

**决定：按主界面启动时开始**

- 用户在设置页编辑好定时值 → 保存到暂存区（与温湿度目标同理）
- 回到主界面，按下 **启动** 按钮 → 定时器开始倒计时
- 如果定时值为 `00:00:00:00`（未设置），启动时不启动定时器

**当前行为变更：**
- 原来：编辑完最后一位按确认即启动
- 改为：仅暂存，启动按钮才触发

## 2. 超时行为

**决定：三者同时发生**

| 行为 | 说明 |
|------|------|
| ✅ 停止加热/风扇 | 关闭所有继电器，`control_heater_set(0)` + `control_fan_set(0)` |
| ✅ 蜂鸣器报警 | 调用 `buzz_play()` 持续响或间歇响，提示用户 |
| ✅ 屏幕提示 TIME UP | 保留当前 `"  TIME UP!   "` 显示 |

附加行为：
- `main_is_running` → 0（系统回到待机状态）
- 主界面启动按钮图标恢复为"启动"状态

## 3. 超时状态复位

**决定：进入定时编辑时自动复位**

- 当前行为不变：在设置页选中"定时"项，按确定进入编辑 → 自动清零 `g_timer_timeout`
- 编辑值保留上次设置值（用户可在此基础上修改）

## 4. 主屏显示

**决定：不需要**

- 主界面不显示倒计时
- 仅在设置页显示定时器状态（IDLE / 倒计时 / TIME UP）

## 5. 与启动/停止按钮的交互

| 场景 | 行为 |
|------|------|
| 未设置定时，按启动 | 系统运行，定时器不启动（`timer_state = IDLE`） |
| 已设置定时，按启动 | 系统运行，定时器开始倒计时 |
| 运行中，按停止 | 系统停止，定时器也停止（`timer_state = IDLE`） |
| 超时后，按启动 | 系统重新运行，且用暂存值重新开始定时 |

## 6. 实现要点

1. **暂存定时值**：在 `GuiTask.c` 添加 `staging_timer_sec` 变量，退出设置时保存
2. **启动时触发**：`MAIN_SELECT_START` 中 `control_set_targets()` 之后启动定时器
3. **超时处理**：在 RTC handler 的 timer 超时分支中，调用 `control_heater_set(0)` + `control_fan_set(0)` + `buzz_play()`
4. **停止时清定时**：按停止时 `timer_state = IDLE`，取消定时

## 7. 涉及文件

| 文件 | 改动 |
|------|------|
| `User/Gui/GuiTask.c` | 暂存逻辑、启动触发、超时联动、停止清定时 |
| `User/Hardeware/Control/control.c` | 可能暴露 `control_stop()` 接口或外部直接调用 `control_heater_set/fan_set` |
